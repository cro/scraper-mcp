This error indicates your GitHub Action (specifically the attestation step) failed due to insufficient repository permissions for the `GITHUB_TOKEN` used in the 
workflow[1][2][3]. The Action needs explicit permissions to persist the attestation (for provenance and security verification).

### How to Fix

1. **Check Workflow Permissions Block**  
   Add the following to your workflow YAML file (if not present) to grant the necessary rights:
   ```yaml
   permissions:
     contents: write
     actions: write
     attestations: write
     id-token: write
   ```
   This block should appear at the top level or for the specific job running the attestation step[3][4][5].

2. **Re-run the Workflow**  
   After updating the permissions block, commit/push your changes and re-run the workflow. This will allow your attestation step to access the repository and create
 provenance metadata correctly[2][3].

3. **Other Tips**  
   - Make sure you’re not running the workflow on a fork or with an incorrectly scoped token[1][6].
   - Check the workflows tab under repository settings for additional permission toggles.
   - If you recently recreated or renamed the repository, disable then re-enable GitHub Actions in the repo’s integration settings to refresh token access[7][8].

⏺ # Scraper MCP

  A context-optimized Model Context Protocol (MCP) server for efficient web scraping. Provides AI tools with pre-processed, filtered web content—reducing token
  usage by 70-90% through server-side CSS filtering and markdown/text conversion.

  ## Quick Start

  ```bash
  docker pull cotdp/scraper-mcp:latest
  docker run -p 8000:8000 cotdp/scraper-mcp:latest

  Access the dashboard at http://localhost:8000

  Features

  - 4 Scraping Tools: Raw HTML, Markdown, Plain Text, Link Extraction
  - CSS Selector Filtering: Extract only relevant content server-side
  - Smart Caching: 3-tier cache system with configurable TTLs
  - Retry Logic: Exponential backoff for transient failures
  - Real-time Dashboard: Monitor requests, cache metrics, and test tools
  - Multi-platform: Supports linux/amd64 and linux/arm64

  Usage with Docker Compose

  services:
    scraper-mcp:
      image: cotdp/scraper-mcp:latest
      ports:
        - "8000:8000"
      environment:
        - TRANSPORT=streamable-http
        - HOST=0.0.0.0
        - PORT=8000
      volumes:
        - cache:/app/cache
  volumes:
    cache:

  MCP Client Configuration

  Connect from Claude Desktop or any MCP client:

  {
    "mcpServers": {
      "scraper": {
        "url": "http://localhost:8000/mcp"
      }
    }
  }

  Available Tags

  - latest - Latest stable release
  - 0.1.0, 0.1, 0 - Semantic version tags
  - main-<sha> - Latest main branch builds

  Documentation

  Full documentation: https://github.com/cotdp/scraper-mcp

  License

  MIT License
  ```
